# Git workflow

- All code changes must be made on a feature branch (never commit directly to `main`).
- Open a PR from the feature branch into `main` and merge it via the PR — do not push to `main` directly.
- Branch naming: `feature/<short-description>` (e.g. `feature/tooltip-enhancements`).
- Squash-merge PRs; delete the branch after merging.
